TallyKeep — Home page (final)

Final home page reflecting all decisions from the iteration phase. Two states of the same single page: empty (first visit) and populated (regular daily view). See design_decisions.md §4 for the full rationale.
Changes from previous version:
  • Sats/BTC unit cycle: click-to-cycle pill instead of segmented toggle (less visual weight)
  • Currency consolidation: single dropdown, empty = no consolidation. No separate on/off toggle.
  • Performance metrics removed: no "↑ 1.2% today" — banking-first, not trading-first
  • Empty state simplified: Getting Started checklist removed. One quiet line + four Add affordances.

Note on the unit cycle pill

How the unit cycle works

The sats ↻ pill in the controls bar is a click-to-cycle. It rotates through:
  1. sats (default, native unit)
  2. BTC (decimal Bitcoin)
  3. fiat currency — only available if a currency is selected in the dropdown to the right
Each click rotates to the next unit. When fiat is "Off" in the dropdown, the cycle goes sats → BTC → sats. When fiat is set to USD, the cycle goes sats → BTC → USD → sats.

The fiat consolidation in the hero subtitle and Holdings table is independent of the cycle — it shows whenever a currency is selected, regardless of which unit the user is currently cycling on.
Reference — unit cycle behavior
Two related but distinct controls. The dropdown turns fiat display on/off. The unit cycle pill chooses which is primary. They cooperate but don't duplicate.
