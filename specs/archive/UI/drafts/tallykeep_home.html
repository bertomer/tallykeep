TallyKeep — Home page

The same single page in two states. The empty state isn't a separate onboarding screen — it's the home page on its first visit, with a guiding overlay that fades as the user fills it in. Once even one Holding is added, the user is in the populated state from then on.
Design decisions baked in:
  • Sats by default, with BTC and (optional) fiat as toggleable views
  • Currency consolidation off by default, opt-in via a "Show fiat values" toggle. When on, user picks the currency.
  • Unit + currency controls live in a small bar above the balance — visible but not noisy
  • Empty state is the same page, with a "Getting started" checklist and the same Add affordances that exist in the populated state's "Add Holding" section
