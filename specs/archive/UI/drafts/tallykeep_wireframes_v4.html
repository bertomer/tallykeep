TallyKeep wireframes — v4

Iteration on the v3 set. This file shows the new onboarding flow replacing v3's single "Profile selection" screen. Other screens (Home, Holdings, Send, Receive, Trading, Sweep, Blueprint) are unchanged from v3 and not redrawn here — refer to v3.
Changelog vs v3:
  • Replaced "Profile selection" screen with a 2-question onboarding (+ optional 3rd) and a setup-review screen
  • Dropped Beginner / Intermediate / Sovereign naming. The configuration emerges from answers; no preset identity.
  • Sidebar profile chip now reads "Custom setup" instead of "Intermediate"
See spec_amendments.md for the rationale and flag-derivation tables.

Where is your Bitcoin currently?

Pick everything that applies. Skip what doesn't. We'll show you only the parts of TallyKeep you actually need.
A custodial exchange — Kraken, Bitstamp, Coinbase, Swissquote…
We'll connect to it via API to read your balance and trigger withdrawals to your own wallets. Your keys stay on the exchange; we just orchestrate the moves.
A software wallet on my phone or laptop
A daily-use wallet you sign with. We'll import the watch-only descriptor (or xpub) — the keys never leave your device.
A hardware wallet — Coldcard, Trezor, Ledger, Jade…
We'll import the watch-only descriptor and generate PSBTs for you to sign on the device.
A multisig setup
Multi-key wallet, possibly with timelocks or geographic separation. Note: v1 stores Vault metadata but only single-key descriptors are accepted on-chain. Full multisig support arrives in v2.
I don't have any Bitcoin yet
We'll show you how to acquire some via a connected exchange. TallyKeep manages Bitcoin you have — it doesn't broker the buy itself.
Onboarding 1 — "Where is your Bitcoin?"
Multi-select. Each checkbox derives one or more feature flags (see spec_amendments.md table). Three checked here would enable Account, Purse, Strongbox, Trading, and Trading→Sweep policy. The user only sees Holding types they actually have. Multisig has its v1-limitation footnote inline.

How much detail do you want to see?

This is about the UI, not your skill. You can change it anytime.
Just the essentials
Balances, send, receive, transactions. Hide UTXOs, derivation paths, raw transaction data, and the technical bits.
Standard view recommended
Everything in Essentials, plus UTXO listing, hygiene flags, descriptor expressions, and the security analyzer.
Show me everything
Everything in Standard, plus coin control on send, custom fee rates (sat/vB), raw transaction hex, replace-by-fee, dry-run sweep policies, and the API documentation link.
Onboarding 2 — "How much detail?"
Single-select, default Standard. The labels describe the UI ("how much you see"), not the user. Each option lists what it adds vs. the previous tier — so the user understands the trade-off concretely. This question alone derives the verbosity flags (UTXO views, raw tx, descriptor, coin control, custom fee rate, RBF, etc.).

Chain-analysis privacy

TallyKeep can warn you about transactions that leak privacy (address reuse, dust UTXOs, round-number outputs). How proactive should it be? Skip if you're not sure.
Yes, flag privacy issues actively
Blueprint section in nav, hygiene warnings shown inline on every Holding. Recommended if you care about not being clustered by chain analysis.
Show me if I ask default
Blueprint available in nav but not foregrounded. Hygiene info shown when you open a UTXO detail.
Don't bother me about it
Hide the Blueprint section entirely. You can re-enable in Settings if you change your mind.
Onboarding 3 — Privacy preference (optional)
Skippable. Defaults to "Show me if I ask" — sensible middle. Drives analysis.blueprint.shown in nav prominence and inline hygiene warnings.
